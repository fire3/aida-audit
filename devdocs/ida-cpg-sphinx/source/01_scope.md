# 1. 范围与交付物（V1）

## 1.1 V1 目标

V1 必须交付可运行的端到端流水线：

- 在 IDA Pro + Hex-Rays 环境中，对二进制中的每个函数生成 MicroCode 并导出结构化数据包（离线可重放）。
- 在纯 Python 环境中，读取导出包并构建 CPG（属性图），满足固定的图不变式与 ID 规则。
- 实现可执行的扫描器框架，并交付一组可验证的 CWE 扫描器（每条结果必须包含可回到 IDA 的定位证据）。

## 1.2 V1 非目标

V1 明确不做以下能力（后续迭代再引入）：

- 全程序精确跨过程指针/别名分析。
- 完整源级 AST 等价重建。
- 重型图数据库（Neo4j/JanusGraph 等）作为必选依赖。
- SMT 级路径可行性求解（V1 仅做轻量模式识别与支配关系近似）。

## 1.3 V1 交付物清单

V1 的“可用”以产物为准：

- **导出包**：一个目录，包含 `meta.json` 与 `functions.jsonl` 等固定文件（见第 3 章）。
- **CPG 内存图**：以 `networkx.MultiDiGraph` 作为默认后端，并提供最小查询 API（见第 6 章）。
- **扫描输出**：`findings.jsonl`（每行一条 finding，字段固定；见第 8 章）。
- **一致性校验**：构建完成后对图结构、地址索引、角色字段做强制校验（见第 6 章）。

## 1.4 V1 可验收标准

- 对一个中等规模二进制（~1k 函数）：
  - 导出包可在无 IDA 的环境中离线构建出 CPG。
  - 任意 `CallSite` finding 都能定位到 `func_ea` 与 `sink_ea`，并能提供 `path_ea`（可截断）与关键节点列表。
- 扫描器至少覆盖：
  - CWE-121/122（危险拷贝/长度可控拷贝）
  - CWE-134（格式化字符串）
  - CWE-78（命令执行）
